프로젝트 개요



여러분은 어딘가로 놀러갈 계획을 세울 때 어떻게 시작하시나요? 일반적으로 주변 사람들의 후기나 계획을 듣고 따라 세우는 경우가 많을 것 입니다. 왜냐하면 어디에 무엇이 있는지 정리된 사이트가 적기 때문입니다. 그렇다고 포털 사이트에 검색하자니 스팸성 광고로 가득차있어 객관적인 정보를 얻기도 힘듭니다. 

이렇듯 정보의 바다의 시대입니다. 새로운 정보보다 잘 정리된 정보가 필요한 시기입니다. 이에 맞추어 ​놀로​라는 서비스는 놀러의 사투리형으로서 놀러갈 곳을 잘 정리해서 보여주는 서비스입니다.


  • 어떤 사용자와 과업을 대상으로 하는가?
 놀러가고 싶은 사용자와 놀러갈 곳을 찾는 검색을 대상으로 한다.
  • 어떤 데이터를 활용해야 하는가?

관광 데이터와 공연 전시 데이터를 활용할 계획이다.

  • 어떤 서비스를 제공하려 하는가?

장소기반, 위치기반 그리고 키워드 기반 검색을 통해 행사와 전시 그리고 숙박 시설에 대한 일반적인 정보, 약도(길 안내), 개장 시간 등과 함께 이미지를 제공한다.


사용자 과업 분석



동영상에서 보여드린 바와 같이 기존 국내 최대 포털사임에도 불구하고 사용자가 원하는 검색 결과는 나오지 않는 것을 알 수 있습니다. 현재 주변 놀러갈 곳은 물론 유명한 지역이라고 할지라도 체계적이고 객관적인 정보는 얻기 힘듭니다. 이를 공공 데이터 기반 API를 활용하여 아래 3가지 장점으로 서비스를 개발하려 합니다.

  1. 신뢰성
  2. 다양성
  3. 객관성

공공 개방 데이터 소개





놀로​가 이용하는 공공 개방 데이터 API는 한국 관광 공사가 제공하는 관광정보 개방 서비스인 TourAPI 3.0입니다. (www.visitkorea.or.kr) 약 5만 8천여건의 관광 정보를 제공하며 다음과 같이 8개국어를 지원하고 있습니다.

  1. 한국어
  2. 영어
  3. 일어
  4. 중국어
  5. 독일어
  6. 프랑스어
  7. 스페인어
  8. 러시아어

지역, 위치 그리고 키워드 기반 관광정보 조회가 가능하며 관광정보의 종류로서는 다음과 같이 8개가 존재합니다.

  1. 일반 관광지
  2. 문화시설
  3. 행사/공연/축제
  4. 여행코스
  5. 레포츠
  6. 숙박
  7. 쇼핑
  8. 음식점
3번의 경우 시작 날짜와 끝 날짜로 검색이 가능하며 6번 숙박의 경우 한옥의 여부도 알 수 있습니다.

또한 관광정보로서 다음과 같은 정보를 제공합니다.
  1. 제목
  2. 연락처
  3. 주소
  4. 좌표(길안내 등)
  5. 개요정보
  6. 휴무일
  7. 개장시간
  8. 주차시설
마지막으로 각 관광 장소의 이미지까지 지원합니다.

스토리보드



사용자 과업 분석에서 조사했듯이 사람들은 놀러갈 곳을 찾을 때 두 가지의 종류로 나뉩니다.

  1. 내 주변
  2. 그 외
따라서 내 주변에서 놀러갈 곳 찾는 기능과 다양한 검색 옵션을 제공하여 위치기반, 장소기반, 키워드 기반으로 검색을 할 수 있도록 지원할 것입니다. 따라서 이에 맞춰 두 버튼을 첫 화면에 배치하였습니다. (1번)

놀러갈 곳들은 사용자에게 가장 익숙하며 스크롤 하기에 편한 Facebook의 Timeline 형태로 제공됩니다. 이는 페이지 단위가 아닌 스크롤이 끝까지 내려지면 자동으로 로딩됨으로서 페이지 전환의 기다림이나 검색 흐름의 끊김등을 방지 할 수 있습니다. (2번)

검색 목록에서 원하는 놀러갈 곳을 클릭했다면 그 장소에 대한 세부 설명이 제공됩니다. (3번)

사용성 평가 - 실험 계획



실험에 앞서 다음과 같은 과업을 정해 실험을 계획하였습니다.

  • 대중교통을 이용해 가족들과 놀러갈만한 장소
  • 여름 휴가를 맞아 놀러갈만한 장소
그리고 각 과업에 기대한 항목은 다음과 같습니다.
  • 대중교통을 이용해 가족들과 놀러갈만한 장소
가족들과 간단한 나들이용으로 갈만한 장소를 찾는데 대중교통을 이용하고 싶을 때를 위해 ​지하철역 기반 검색​을 만들었습니다. 그리고 사용자는 이를 적극 활요하길 원했습니다. ​지하철역 기반 검​색​은 모든 지하철역(640여개)의 GPS를 기반으로 약 1km이내의 놀러갈만한 장소를 검색합니다.
  • 여름 휴가를 맞아 놀러갈만한 장소
여름 휴가를 맞이하여 바다나 강가를 찾는데 대게 네이버나 구글을 통해 찾습니다. 하지만 항상 나오던 곳만 나오고 더 자세한 정보를 체계적인 기준에 따라 얻기 힘듭니다. 이를 보완한 검색으로 ​키워드 기반 검색​을 만들었습니다. 원하는 키워드를 입력하여 검색을 하면 해당 키워드가 제목에 포함된 놀러갈만한 장소를 검색합니다.

그리고 실험을 통해 다음과 같은 항목을 평가하였습니다.
  1. 검색 속도 - 랙이 걸리진 않는지
  2. 정보의 정확성 및 다양성 - 제공된 정보가 부족하진 않은지
  3. 검색 결과의 양과 질 - 검색 결과가 부족하거나 부실하지 않은지
  4. 이용 편리성 - 이용하는데 불편하거나 헤매진 않는지
각 부분을 5점 만점으로 진행하였습니다.

결론 및 향후 개선방향



그동안 여러 프로젝트를 해오면서 사용자 입장에서의 객관적이고 증명된 도구를 사용한 사용자 경험을 조사해본 적이 없었습니다. 그래서 완성된 프로젝트들은 다른 사람들이 보기에 완성도가 떨어졌고 큰 호응을 얻지 못했었습니다. 그러나 이번 Human Computer Interface 수업을 들으며 배운 여러가지 사용자 경험 조사 도구를 이용해 이 프로젝트의 완성도와 UX(User Experience)를 높이는 방법을 알게 되었습니다.


이번 사용자 경험 조사를 통해 알게 된 UX 향상 방안은 다음과 같습니다.

  1. 장소 세부 정보 페이지에서 마우스를 내릴 때 지도에 줌(Zoom) 기능을 함
  2. 무엇을 어떻게 검색할 수 있을지 가이드 라인이 필요함
  3. 일부 안드로이드 기기와 아이폰 미지원
개발자 입장에서는 환경의 제약이나 가치관의 제약으로 지나칠 수 있는 부분이었으나 충분한 사용자 경험 조사를 통해 완성도를 높일 수 있는 피드백을 구체적으로 받게 되어 너무 기분이 좋고 더 잘할 수 있을 것 같습니다. 정말 배운게 많고 그것을 앞으로 사용하게 될 유용한 배움이어서 더 기분이 좋습니다.
